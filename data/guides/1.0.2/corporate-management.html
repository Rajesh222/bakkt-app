Overview

Corporate management enables businesses to onboard as entities on the Bakkt platform. This guide covers corporate creation, user management, and KYB (Know Your Business) verification.

Corporate Status Lifecycle

Corporate entities progress through KYB (Know Your Business) verification:


flowchart LR
    CREATED --> KYB_PENDING --> ACTIVE
    KYB_PENDING --> REJECTED
StatusDescriptionNext Steps / Considerations
CREATEDCorporate entity successfully created in systemObtain the SumSub URL to complete the verification viaGET /corporate/{corporate_uuid}/kyb/applicant/url) and share it with the Corporate Administrator
KYB_PENDINGKYB verification in progressKYB verification process may take some time. Keep the Corporate Administrator informed of the pending status
ACTIVEKYB verification successful, fully activeOnly with ACTIVE status entity is allowed to transact
REJECTEDKYB verification rejected by complianceSuch decisions are generally final, you’re welcome to reach out to our compliance team if you wish to request further clarification or raise an appeal

Create Corporate Entity

Create a new corporate entity on the platform

Endpoint

POST /corporate

Required Fields

  • legal_name Official registered business name
  • type Business structure type
  • contact_details Primary contact information
  • registered_address Legal registered address
  • target_address Ethereum address for crypto operations

Request Example

const response = await fetch('https://api.bakkt.com/corporate', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    legal_name: 'Acme Corporation Ltd',
    type: 'LIMITED_LIABILITY',
    contact_details: {
      name: 'John Doe',
      email: 'john@acme.com',
      phone: '+447871556837'
    },
    registered_address: {
      address_line_1: '123 Business Street',
      post_code: 'SW1A 1AA',
      city: 'London',
      country: 'GB'
    },
    target_address: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb',
    accepted_terms_of_service_hash: 'base64_hash_here'
  })
});

const corporate = await response.json();
console.log('Corporate UUID:', corporate.corporate_uuid);
console.log('Status:', corporate.status); // 'CREATED'

Business Types

Supported corporate types:

  • LIMITED_LIABILITY Limited Liability Company
  • SOLE_TRADER Sole Proprietorship
  • PARTNERSHIP Partnership
  • PUBLIC_LIMITED_COMPANY Public Limited Company
  • JOINT_STOCK_COMPANY Joint Stock Company
  • CHARITY Charitable Organization

Add User to Corporate

Link an individual user to a corporate entity with a specific role

Endpoint

POST /corporate/user

Request Example

const response = await fetch(`https://api.bakkt.com/corporate/${corporateUuid}/user`, {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_uuid: 'user-uuid-here',
    role: 'ADMINISTRATOR'
  })
});

Available Roles

Currently supported role:

  • ADMINISTRATOR Full access to corporate operations

Future Roles
Additional roles (VIEWER, OPERATOR, etc.) may be added. Check the API reference for the latest supported roles

Unlink User from Corporate

Remove a user's association with a corporate entity.

Endpoint

DELETE /corporate/user

Request Example

const response = await fetch(
  `https://api.bakkt.com/corporate/${corporateUuid}/user/${userUuid}`,
  {
    method: 'DELETE',
    headers: {
      'Authorization': 'YOUR_API_KEY',
      'unblock-session-id': sessionId
    }
  }
);

// Returns 204 No Content on success
❗️

Admin Required
This endpoint can only be called by a user with the ADMINISTRATOR role in the specific corporate entity

Complete Corporate Onboarding Workflow

// Step 1: Create corporate entity
const corporate = await bakkt.createCorporate({
  legal_name: 'Acme Corporation Ltd',
  type: 'LIMITED_LIABILITY',
  contact_details: {
    name: 'John Doe',
    email: 'john@acme.com',
    phone: '+447871556837'
  },
  registered_address: {
    address_line_1: '123 Business Street',
    post_code: 'SW1A 1AA',
    city: 'London',
    country: 'GB'
  },
  target_address: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb'
});

console.log('Corporate created:', corporate.corporate_uuid);

// Step 2: Create admin user (if doesn't exist)
const adminUser = await bakkt.createUser({
  first_name: 'John',
  last_name: 'Doe',
  email: 'john@acme.com',
  country: 'GB',
  target_address: '0x...'
});

// Step 3: Link admin to corporate
await bakkt.addUserToCorporate(corporate.corporate_uuid, {
  user_uuid: adminUser.user_uuid,
  role: 'ADMINISTRATOR'
});

// Step 4: Get KYB form URL
const kybUrl = await bakkt.getKYBFormUrl(corporate.corporate_uuid);
console.log('Send this URL to admin:', kybUrl.url);

// Step 5: Admin completes KYB form externally
// Bakkt sends webhook when KYB is approved

// Step 6: Monitor KYB status via webhook or polling
webhook.on('KYB ACTIVE', (data) => {
  console.log('Corporate is now active!');
  // Corporate can now use all Bakkt services
});

Best Practices

🏢

Follow guideline to ensure a smooth corporate creation process.

  • Corporate Setup: Require admin user creation first

Clearly communicate KYB requirements upfront

Set both registered and trading addresses if different

Link all relevant users to corporate entity

  • User Management: Create individual users before linking to corporate

    Assign appropriate roles based on responsibilities

    Document who has admin access for compliance

    Review and audit user access regularly

  • Address Management

    Registered Address: Legal address on company documents

    Trading Address: Day-to-day operations address (can be different)

    Both addresses should be complete and accurate for KYB

Use Case: Multi-Corporate User

A single user can be linked to multiple corporate entities:

// User is admin for multiple companies
const user = await bakkt.getUser(sessionId);

console.log('Linked corporates:', user.linked_corporates_uuid);
// ['corp-1-uuid', 'corp-2-uuid', 'corp-3-uuid']

// Switch context by using different corporate_uuid in API calls
await bakkt.getCorporateDetails('corp-1-uuid');
await bakkt.getCorporateDetails('corp-2-uuid');
