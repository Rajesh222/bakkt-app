🚀 Feature 1: On-Ramp (Fiat to Stablecoin)

Link a Accounts API Virtual IBAN to a crypto wallet for automatic fiat-to-stable conversions.

How On-Ramp Works

Note: This section includes a sequence diagram showing the workflow. Refer to the original documentation for the visual diagram.

Prerequisites

1. Accounts API Virtual IBAN

Create a Virtual IBAN in the user's preferred currency:

  POST /user/bank-account/bakkt
  {
    "currency": "EUR"
  }

See Accounts API guide for details.

2. Target Address (from Onboarding)

User must have target_address set during creation in Onboarding API. This becomes the DEFAULT destination for on-ramp conversions.

The address should be the user's external wallet (MetaMask, Coinbase, etc.).

3. Token Preferences (Optional)

Set preferred stablecoin and chain. Defaults to USDC on Polygon if not set.

PATCH /user/token-preferences
{
  "currency": "EUR",
"chain": "polygon",
  "token": "USDC"
}

User On-Ramp Example

Complete flow for individual users:

// Assumes: User created in Onboarding with FULL_USER status and target_address set

// 1. Create Virtual IBAN in Accounts API
const virtualAccount = await fetch('https://api.bakkt.com/user/bank-account/bakkt', {
  method: 'POST',
  headers: {
    'Authorization': apiKey,
    'unblock-session-id': sessionId
  },
  body: JSON.stringify({ currency: 'EUR' })
});

const va = await virtualAccount.json();
console.log('Virtual IBAN:', va.iban);
console.log('VA UUID:', va.uuid);

// 2. Link Virtual IBAN to On-Ramp wallet
const wallet = await fetch(`https://api.bakkt.com/user/wallet/polygon?linked_bank_account_uuid=${va.uuid}`, {
  headers: {
    'Authorization': apiKey,
    'user-uuid': userUuid
  }
});

const walletData = await wallet.json();
console.log('Conversion active - fiat deposits will convert to USDC');
console.log('Destination wallet:', walletData[0].address); // User's target_address

// 3. User deposits EUR to Virtual IBAN
// User transfers from their bank to va.iban

// 4. Automatic conversion happens
// → Fiat received in VA
// → Auto-converts to USDC
// → Sends to user's target_address
// → Webhook: fiatToCrypto SUCCESS

Corporate On-Ramp Example

Same flow for corporates:

// 1. Create Corporate Virtual IBAN (Accounts API)
POST /corporate/{corporate_uuid}/bank-account/bakkt
{
  "currency": "EUR"
}

// 2. Link to corporate wallet (On/Off Ramp API)
GET /corporate/{corporate_uuid}/wallet/polygon?linked_bank_account_uuid={va_uuid}

// Returns corporate's target_address from Onboarding

Info: On-Ramp Benefit: Users just deposit fiat to an IBAN like any bank transfer. Conversion to crypto happens automatically in the background.
