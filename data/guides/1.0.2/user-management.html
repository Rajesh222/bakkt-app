Overview

User management is the foundation of onboarding individuals to the Bakkt platform. This guide covers the complete user lifecycle from creation and authentication and later through verification and profile updates.

User Status Lifecycle

Users progress through different statuses as they complete verification:

flowchart LR
A[PHONE_VERIFICATION_NEEDED *US only] --> B[KYC_NEEDED]
B --> C[PENDING_KYC_DATA]
C --> D[KYC_PENDING]
D --> E[FULL_USER]
D --> F[SOFT_KYC_FAILED]
D --> G[HARD_KYC_FAILED]
StatusDescriptionNext Steps / Considerations
PHONE_VERIFICATION_NEEDED*Phone verification required only for U.S. usersSend OTP to the user's phone number
KYC_NEEDEDUser must complete KYC to transactPrompt user to start KYC
PENDING_KYC_DATAUser started KYC, documents are missingRemind user to submit documents. Check missing docs via GET /user/kyc/applicant
KYC_PENDINGKYC verification under reviewWait for the review to complete
FULL_USERFull KYC approved; User fully verifiedOnly with FULL_USER status user is allowed to transact
SOFT_KYC_FAILEDKYC failed; User can retry after correcting dataCheck webhook data for reasons. Guide user to correct and retry
HARD_KYC_FAILEDKYC permanently failed; User cannot be servedInform user of KYC failure. No retry possible
SUSPENDEDUser suspended, based on the compliance decision. User cannot take any action in the systemFor further clarity contact our support

Core user operations

Create a user

  • Create a new user account with required per country inputs
  • Endpoint POST /user
  • Request Example:
const response = await fetch('https://api.bakkt.com/user', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    first_name: 'Alice',
    last_name: 'Smith',
    email: 'alice@example.com',
    country: 'GB',
    target_address: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb',
    accepted_terms_of_service_hash: 'base64_hash_here'
  })
});

const user = await response.json();
console.log('User UUID:', user.user_uuid);
console.log('Status:', user.status); // 'KYC_NEEDED'

Update user profile

  • Update user profile details, addresses, and preferences

  • Most fields are updatable for users who do not have FULL_USER status. Once KYC is complete, only limited fields can be changed. If update required on restricted parameters, please contact our support team

    Parameters restricted after KYC: first_name, last_name , date_of_birth , email , phone number, SSN , BVN

  • Endpoint PATCH /user

  • Request Example:

const response = await fetch('[https://api.bakkt.com/user](https://api.bakkt.com/user)', {
method: 'PATCH',
headers: {
'Authorization': 'YOUR_API_KEY',
'unblock-session-id': sessionId,
'Content-Type': 'application/json'
},
body: JSON.stringify({
target_address: '0x_NEW_ADDRESS',
target_solana_address: 'NEW_SOLANA_ADDRESS'
})
});

Target Address

📘

Target Address: This is the user's EXTERNAL wallet (MetaMask, Coinbase, hardware wallet, etc.) where on-ramp conversions will be sent by default. This is different from Bakkt custody addresses.

  • For EVM chains: Set target_address (Ethereum address format: 0x...)
  • For Solana: Set target_solana_address (Solana address format)
  • For multi-chain support: Set both addresses

Target Address Types

Specify whether the user's wallet is self-hosted or hosted by a VASP:

  • SELF_HOSTED: User controls their own private keys (MetaMask, hardware wallet)
  • HOSTED: Hosted by exchange or custodian (Coinbase, Binance, etc.)

If hosted, provide the VASP name using target_address_vasp field.

Phone Verification

🇺🇸

This step is required for US users only

Send OTP

  • Send OTP (one-time password) verification code to the user
  • Endpoint POST /user/verification/phone/send
  • Request Example:
await fetch('https://api.bakkt.com/user/verification/phone/send', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'unblock-session-id': sessionId
  }
});
// OTP sent to user's registered phone/email

Verify OTP

Verify OTP

  • Verify OTP code provided by the user
  • Endpoint POST /user/verification/phone/check
  • Request Example:
const response = await fetch('https://api.bakkt.com/user/verification/phone/check', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'unblock-session-id': sessionId,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    code: '123456'
  })
});

const result = await response.json();
console.log('Status:', result.status); // 'approved', 'pending', or 'rejected'

Best Practices

💪

Frequent user creation issues & how to prevent them

  • Email or phone number are not unique

Each account must have a unique email address

For US users, both the email address and phone number must be unique before creating a user

  • Required target addresses are missing

Always set target_address during user creation (required for crypto flows)

For Solana support, also provide target_solana_address

  • Terms and Conditions Hashes are missing

Collect and store: Terms and Conditions, Terms of Service and Privacy Policy hashes during user creation to meet compliance requirements.

🏦

Account Status Management

  • Account Suspended

The account is restricted following a compliance review. The user cannot transact or update data. If the restriction is lifted, re-KYC is not required.

A webhook is sent to the merchant upon status change.

Contact support for clarification or appeal.

  • Account Deactivated (vIBAN)

User IBANs may be deactivated after 12 months of inactivity.

Merchants are notified in advance and receive a list of affected vIBANs.

Account deactivation is final; reopening requires a new KYC process completed by the user.

Complete User Onboarding Workflow

End-to-end example:

// 1. Create user
const user = await bakkt.createUser({
  first_name: 'Alice',
  last_name: 'Smith',
  email: 'alice@example.com',
  country: 'GB',
  accepted_terms_of_service_hash: 'base64_hash_here'
});

console.log('User created:', user.user_uuid);

// 2. Authenticate (see Authentication guide)
await bakkt.triggerOTP(user.user_uuid);
const session = await bakkt.submitOTP(user.user_uuid, '123456');

console.log('Authenticated:', session.session_id);

// 3. Initiate KYC (see KYC/KYB guide)
await bakkt.createKYCApplication(session.session_id);
const sumsubToken = await bakkt.getKYCToken(session.session_id);

// 4. User completes KYC
// (Using Sumsub Web SDK with the token)

// 5. Monitor status
const status = await bakkt.getUser(session.session_id);
console.log('Current status:', status.status);

// 6. Once FULL_USER, can use other APIs
if (status.status === 'FULL_USER') {
  // Ready for Accounts API, Stablecoin API, etc.
}
