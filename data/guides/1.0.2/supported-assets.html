Token Explanations

USDC (Native)

Native USDC issued by Circle directly on the blockchain.

**Available on**: All chains

**Best for**: Most common and liquid stablecoin, recommended for most use cases

USDC.e (Bridged)

Bridged USDC - Circle's USDC that was bridged from Ethereum to other chains.

**Available on**: Ethereum, Polygon, Optimism, Solana, BSC, Avalanche

**Note**: Being phased out in favor of native USDC on most chains

USDT (Tether)

Tether USD stablecoin.

**Available on**: Ethereum, Polygon, Optimism, Arbitrum, Tron, Base, BSC, Avalanche

**Best for**: High liquidity, widely accepted on exchanges

DAI

DAI stablecoin - Decentralized stablecoin by MakerDAO.

**Available on**: Ethereum, Polygon, Optimism, Arbitrum

**Best for**: DeFi integrations, decentralized applications

Euro Stablecoins

cEUR: Circle EUR stablecoin (Arbitrum, Celo)

**agEUR**: Angle Protocol EUR stablecoin (Ethereum)

**Best for**: EUR on-ramp and off-ramp operations in Europe

cUSD

Celo USD stablecoin native to Celo blockchain.

**Available on**: Celo

**Best for**: Celo ecosystem integrations, mobile-first applications

Understanding "Receiving Address"

When you create a destination bank account, Bakkt returns a receiving_address:

{
  "uuid": "bank-account-uuid",
  "receiving_address": "0xf7571e12301E70aCb89D5ACf0258F55c88AD097f"
}

Info:
What is this? The receiving_address is a dedicated crypto wallet address linked to this specific bank account. When users send crypto to this address, it automatically triggers an offramp to the linked bank account.

Benefits:

  • Each bank account gets its own unique crypto address
  • Simplifies offramp flow - send crypto to specific address
  • Automatic routing to correct destination
  • No need to specify bank account in separate API call

Supported Bank Account Types

UK (GBP)

Required Fields:
- currency: "GBP"
- account_number: 8 digits
- sort_code: 6 digits (no hyphens)

```json
{
  "account_details": {
    "currency": "GBP",
    "account_number": "31926819",
    "sort_code": "123456"
  }
}
```

Europe (EUR)

Required Fields:
- currency: "EUR"
- iban: Full IBAN

```json
{
  "account_details": {
    "currency": "EUR",
    "iban": "PT50002700000001234567833"
  }
}
```

Mexico (MXN)

Required Fields:
- currency: "MXN"
- account_number: Bank account (Clabe)
- account_type: Account type
- swift_code: Bank's SPEI
- bank_name: Bank name
- bank_code, bank_subcode: Optional
- beneficiary_address: Beneficiary address
- beneficiary_phone_number: Phone number
- beneficiary_bank_address: Bank address
- recipient_relationship: Relationship to sender
- remittance_purpose: Purpose of transfer

```json
{
  "account_details": {
    "currency": "MXN",
    "account_number": "2982352921",
    "account_type": "Checking",
    "swift_code": "BDEMMXMM",
    "bank_name": "Banxico",
    "bank_code": "SBIN0011821",
    "bank_subcode": "SBIN0011821",
    "beneficiary_address": "123 Main St, Mexico City",
    "beneficiary_phone_number": "+5215512345678",
    "beneficiary_bank_address": "456 Bank St, Mexico City",
    "recipient_relationship": "Self",
    "remittance_purpose": "Salary"
  }
}
```

**Relationship Options**: Self, Friend, Employer, Colleague, Family, Husband, Wife, Father, Mother, Son, Daughter, Brother, Sister

**Purpose Options**: Salary, Real Estate, Savings, Gift, Education Support, Home Improvement

Brazil (BRL)

Required Fields:
- currency: "BRL"
- pix_key: PIX key (email, phone, CPF, or account number)
- tax_id: CPF format
- recipient_relationship: Relationship to sender
- remittance_purpose: Purpose of transfer

```json
{
  "account_details": {
    "currency": "BRL",
    "pix_key": "edee3d60-a20b-4a87-aca5-83d00d6167c9",
    "tax_id": "93659452101",
    "recipient_relationship": "Self",
    "remittance_purpose": "Gift"
  }
}
```

> **Note:**
  **PIX System**: Brazil's instant payment system. PIX key can be email, phone, CPF, or random UUID. CNPJ (company tax ID) is not accepted.

Currency-Specific Examples

Brazil PIX Payment

// Brazil uses PIX for instant payments
const beneficiary = await bakkt.stablecoin.addBeneficiary({
  account_name: 'Brazilian Supplier',
  account_details: {
    currency: 'BRL',
    pix_key: 'supplier@company.com.br', // Can be email, phone, CPF, or UUID
    tax_id: '93659452101', // CPF format
    recipient_relationship: 'Employer',
    remittance_purpose: 'Salary'
  }
});
