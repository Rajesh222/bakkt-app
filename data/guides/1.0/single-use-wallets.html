📦 Single-Use Wallets (Corporate Only)

Generate unique, one-time wallet addresses for batch processing and invoice tracking.

What Are Single-Use Wallets?

Single-use wallets are one-time crypto deposit addresses linked to a specific external reference (like an invoice ID). When crypto is sent to these addresses, it automatically offramps to the corporate's main beneficiary.

Key characteristics:

  • ✅ One address per external reference (e.g., one per invoice)
  • ✅ Automatic offramp when crypto received
  • ✅ Links to corporate's main beneficiary bank account
  • ✅ Perfect for batch processing and reconciliation
  • ⚠️ Whitelisted corporates only (contact sales@bakkt.com)

Use Cases

Invoice Payments

Generate unique wallet per invoice. Track which customer paid which invoice.

Batch Settlements

Process multiple payments simultaneously with unique addresses for reconciliation.

Customer Deposits

Assign each customer a unique address. Auto-convert and settle to treasury.

Vendor Payments

Track vendor-specific crypto payments with external references.

How Single-Use Works

Corporate Example

// Prerequisites: Corporate has ACTIVE status and main beneficiary set

// 1. Create single-use wallet for invoice
const singleUseWallet = await fetch(
  `https://api.bakkt.com/corporate/${corporateUuid}/single-offramp-wallet`,
  {
    method: 'POST',
    headers: {
      'Authorization': apiKey,
      'unblock-session-id': sessionId
    },
    body: JSON.stringify({
      external_reference: 'invoice-12345'
    })
  }
);

const wallet = await singleUseWallet.json();
console.log('Chain:', wallet.chain);
console.log('Address:', wallet.address);
console.log('Reference:', wallet.external_reference);

// 2. Share address with customer
sendInvoice(customer, {
  invoiceId: 'invoice-12345',
  amount: '$1000 or equivalent USDC',
  paymentAddress: wallet.address,
  chain: wallet.chain
});

// 3. When customer pays, auto-offramp happens
// → Webhook received with external_reference: "invoice-12345"
// → Mark invoice as paid in your system
// → Fiat arrives in corporate's main beneficiary account
**Perfect for Reconciliation**: The external_reference in webhooks lets you automatically match payments to invoices without manual reconciliation.
