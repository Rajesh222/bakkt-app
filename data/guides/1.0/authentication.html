Overview

Bakkt supports two authentication methods for user sessions:

  1. SIWE (Sign In With Ethereum) - Web3-native authentication
  2. Email OTP - Traditional one-time password via email

Both methods return a session_id that's used for authenticated API requests.

Authentication Flow

    participant User
    participant App
    participant Bakkt

    alt SIWE Authentication

        Bakkt-->>App: session_id + user_uuid
    else Email OTP

        User->>App: Enters OTP
        App->>Bakkt: POST /auth/otp (user_uuid + OTP)
        Bakkt-->>App: session_id + user_uuid
    end

    App->>Bakkt: Authenticated API requests (session_id)

Method 1: SIWE

Web3-native authentication using wallet signatures.

How SIWE Work

  1. User connects their wallet (MetaMask, WalletConnect, etc.)
  2. Your app generates a SIWE message
  3. User signs the message with their wallet
  4. Your app sends message + signature to Bakkt
  5. Bakkt verifies signature and returns session
How to generate SIWE message

Step 1: Generate SIWE

Use a library like siwe to generate the message:

import { SiweMessage } from 'siwe';

// Create SIWE message
const message = new SiweMessage({
  domain: 'your-app.com',
  address: userWalletAddress, // User's Ethereum address
  statement: 'Sign in to Bakkt',
  uri: 'https://your-app.com/auth/login',
  version: '1',
  chainId: 1, // Ethereum mainnet
  nonce: generateNonce(), // Random nonce for security
  issuedAt: new Date().toISOString(),
  expirationTime: new Date(Date.now() + 4 * 60 * 60 * 1000).toISOString() // 4 hours
});

const messageString = message.prepareMessage();

Step 2: User Signs Message

// Request signature from user's wallet
const signature = await window.ethereum.request({
  method: 'personal_sign',
  params: [messageString, userWalletAddress]
});

Step 3: Authenticate with Bakkt

const response = await fetch('https://api.bakkt.com/auth/login', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    message: messageString,
    signature: signature
  })
});

const session = await response.json();
console.log('Session ID:', session.bakkt_session_id);
console.log('User UUID:', session.user_uuid);

// Store session_id securely for future requests
localStorage.setItem('bakkt_session', session.bakkt_session_id);
SIWE Example
import { SiweMessage } from 'siwe';
import { useAccount, useSignMessage } from 'wagmi';

function SignInButton() {
  const { address } = useAccount();
  const { signMessageAsync } = useSignMessage();

  const handleSignIn = async () => {
    // 1. Create SIWE message
    const message = new SiweMessage({
      domain: window.location.host,
      address,
      statement: 'Sign in to Bakkt',
      uri: window.location.origin,
      version: '1',
      chainId: 1,
      nonce: crypto.randomUUID()
    });

    // 2. Sign message
    const signature = await signMessageAsync({
      message: message.prepareMessage()
    });

    // 3. Authenticate
    const session = await bakkt.siweLogin(message.prepareMessage(), signature);

    // 4. Store session
    setSession(session.bakkt_session_id);
  };

  return <button onClick={handleSignIn}>Sign In with Ethereum</button>;
}
// Using ethers.js
import { ethers } from 'ethers';

async function signInWithEthereum() {
  // Connect wallet
  const provider = new ethers.providers.Web3Provider(window.ethereum);
  await provider.send("eth_requestAccounts", []);
  const signer = provider.getSigner();
  const address = await signer.getAddress();

  // Create message
  const message = `api.bakkt.com wants you to sign in with your Ethereum account:
${address}

Sign in with Ethereum

URI: https://api.bakkt.com/auth/login
Version: 1
Chain ID: 1
Nonce: ${generateNonce()}
Issued At: ${new Date().toISOString()}
Expiration Time: ${new Date(Date.now() + 4*60*60*1000).toISOString()}`;

  // Sign
  const signature = await signer.signMessage(message);

  // Authenticate
  const response = await fetch('https://api.bakkt.com/auth/login', {
    method: 'POST',
    headers: {
      'Authorization': 'YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ message, signature })
  });

  return await response.json();
}

Method 2: Email OTP

Traditional one-time password sent via email.

How to Generate an Email OTP

Step 1: Trigger OTP

const response = await fetch('https://api.bakkt.com/auth/login', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_uuid: 'user-uuid-here'
  })
});

// OTP sent to user's registered email
console.log('OTP sent to user email');

Step 2: Submit OTP

User receives 6-digit code via email and submits it:

const response = await fetch('https://api.bakkt.com/auth/otp/', {
  method: 'POST',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_uuid: 'user-uuid-here',
    one_time_password: '123456'
  })
});

const session = await response.json();
console.log('Session ID:', session.bakkt_session_id);
console.log('User UUID:', session.user_uuid);

// Store session securely
localStorage.setItem('bakkt_session', session.bakkt_session_id);
OTP Example
async function loginWithOTP(userUuid) {
  // 1. Trigger OTP
  await fetch('https://api.bakkt.com/auth/login', {
    method: 'POST',
    headers: {
      'Authorization': API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ user_uuid: userUuid })
  });

  // 2. Show OTP input to user
  const otp = await promptUser('Enter OTP from email:');

  // 3. Submit OTP
  const response = await fetch('https://api.bakkt.com/auth/otp/', {
    method: 'POST',
    headers: {
      'Authorization': API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      user_uuid: userUuid,
      one_time_password: otp
    })
  });

  const session = await response.json();
  return session.bakkt_session_id;
}

Using Session ID

Once authenticated, include the session ID in all requests:

// Making authenticated requests
const response = await fetch('https://api.bakkt.com/user', {
  method: 'GET',
  headers: {
    'Authorization': 'YOUR_API_KEY',
    'bakkt-session-id': sessionId
  }
});

Session Headers

Two ways to pass user identity:

For User-Initiated Actions (POST, PATCH, DELETE):

Authorization: YOUR_API_KEY
bakkt-session-id: SESSION_ID

For Backend Queries (GET operations):

Authorization: YOUR_API_KEY
user-uuid: USER_UUID
💡

Backend Operations:

Use user-uuid header for server-side queries when you don't have a user session.
Use bakkt-session-id for user-initiated actions.

Session Management

Session Expiration

Sessions expire after:

  • Default: 7 days of inactivity
  • Maximum: 30 days from creation

Handling Expiration

async function makeAuthenticatedRequest(endpoint, options = {}) {
  const sessionId = localStorage.getItem('bakkt_session');

  const response = await fetch(endpoint, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': API_KEY,
      'bakkt-session-id': sessionId
    }
  });

  // Handle session expiration
  if (response.status === 401) {
    // Session expired - trigger re-authentication
    const newSessionId = await reAuthenticate();
    localStorage.setItem('bakkt_session', newSessionId);

    // Retry original request
    return makeAuthenticatedRequest(endpoint, options);
  }

  return response;
}

Session Refresh

Currently, Bakkt doesn't support session refresh tokens. When a session expires, users must re-authenticate using SIWE or OTP.

Note: Best Practice: Implement automatic re-authentication flow before session expiry. Prompt users to re-authenticate proactively rather than waiting for 401 errors.

Security Best Practices

SIWE Security

  • Verify Domain: Ensure SIWE message domain matches your app domain

    • Check Nonce: Use cryptographically secure random nonces

    • Validate Expiration: Set reasonable expiration times (2-4 hours)

    • Verify Signature: Let Bakkt verify the signature server-side

    • Address Matching: Ensure signed address matches user's target_address

OTP Security

  • Rate Limiting: Limit OTP requests per user (e.g., 3 per 10 minutes)

    • Attempt Limits: Lock account after 5 failed OTP attempts

    • Expiration: OTPs expire after 10 minutes

    • No Storage: Never store OTPs in logs or databases

    • Secure Delivery: Ensure email channel is secure

Session Security

  • Secure Storage: Use httpOnly cookies or secure storage (not localStorage for sensitive apps)
    • HTTPS Only: Always use HTTPS for API requests
    • Short Expiry: Consider shorter session expiry for high-risk operations
    • Logout: Implement explicit logout to clear sessions
    • Monitor: Log authentication attempts and unusual patterns
